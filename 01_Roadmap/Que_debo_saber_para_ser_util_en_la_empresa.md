# Qué debo saber para ser útil en la empresa

## Capacidades operativas

- Levantar servicios con [[Docker_Compose]].
- Leer logs sin bloquearte.
- Inspeccionar contenedores con `docker exec`.
- Entender variables de entorno y volúmenes.
- Aplicar y diagnosticar patches con [[Git_Apply]].
- Leer un repo grande con [[Comandos_para_buscar_en_repos]].

## Capacidades de IA aplicada

- Convertir requisitos en datos evaluables.
- Medir retrieval con golden sets.
- Saber cuándo usar [[BM25]], embeddings o [[Hybrid_Search]].
- Entender [[Qdrant]] como base vectorial con payloads, no como magia.
- Saber qué hace [[LiteLLM]] frente a [[vLLM]].

## Capacidades de dominio automoción

- Leer CAN: timestamp, arbitration ID, DLC, payload.
- Reconocer UDS: service ID, DID, NRC.
- Entender DoIP como diagnóstico sobre IP.
- Extraer features para anomalías y entrenar [[Autoencoders]].

> [!idea]
> Ser útil pronto no significa saberlo todo. Significa poder localizar el fallo: despliegue, patch, retrieval, modelo, datos o evaluación.
