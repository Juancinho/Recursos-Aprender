# Cómo usar recursos y referencias

Esta bóveda ya tiene apuntes propios. Los recursos externos sirven para tres cosas:

1. **Ver la fuente oficial**: confirmar comandos, configuración y APIs.
2. **Ver otra explicación**: reforzar el modelo mental cuando una nota se quede corta.
3. **Profundizar con papers/cursos**: entender por qué existen las herramientas, no solo cómo se usan.

## Regla de lectura activa

Antes de abrir un recurso, escribe una pregunta concreta:

- "¿Cómo define Docker un volumen?"
- "¿Cómo configura OpenWebUI una base vectorial externa?"
- "¿Qué problema resuelve PagedAttention?"
- "¿Qué campos tiene un frame CAN?"

Después de leer/ver, añade una nota de 5 líneas:

- qué aprendí;
- qué comando o concepto puedo aplicar;
- qué duda queda;
- a qué nota interna se conecta;
- qué práctica haré.

## Tipos de recursos

| Tipo | Cuándo usarlo | Ejemplo |
|---|---|---|
| Documentación oficial | Para comandos, variables, APIs | Docker, Qdrant, OpenWebUI, vLLM |
| Curso/tutorial | Para aprender ordenadamente | Hugging Face Course, Docker workshop |
| Paper | Para entender la idea original | Attention, PagedAttention, BM25 |
| Manual de herramienta | Para uso operativo | Wireshark User Guide |
| Producto/plataforma | Para entender qué vende o promete | Qodo docs |

## Cómo evitar perder tiempo

> [!warning]
> No abras diez pestañas. Abre una, extrae una acción, vuelve al lab.

Checklist:

- [ ] El recurso responde una pregunta concreta.
- [ ] Lo he conectado con una nota interna concreta de la bóveda.
- [ ] He ejecutado algo después.
- [ ] He descartado lo que no necesito ahora.
