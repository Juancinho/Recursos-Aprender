# Bóveda IA Technica

Esta bóveda es tu ruta práctica para ponerte al día en las prácticas de IA en Technica Engineering. Está pensada para estudiar con Obsidian, sin depender de plugins, y mezcla teoría, laboratorios, checklists, comandos y código.

El foco no es aprender IA de forma genérica. El foco es entender un sistema realista de automatización de testing basado en requisitos: [[OpenWebUI]], [[Qdrant]], [[LiteLLM]], [[vLLM]], [[Qwen_Local]], embeddings, BM25, búsqueda híbrida, análisis de repos grandes, patches sobre imágenes oficiales, [[Qodo]], [[LangGraph]], [[MCP]], [[Wireshark]], CAN/UDS/DoIP y autoencoders.

> [!todo]
> Empieza por [[PROXIMOS_PASOS]] y después sigue [[Roadmap_Maestro]]. No intentes estudiar todo a la vez.

## Forma de trabajo

1. Lee la nota conceptual mínima.
2. Ejecuta el laboratorio asociado.
3. Escribe qué entendiste en [[Diario_Practicas_Template]] o [[Diario_Estudio_Template]].
4. Convierte cada duda en una búsqueda concreta en el repo, en Docker o en logs.

## Entregables personales recomendados

- Un `MAPA_REPO.md` de cualquier repo grande que te asignen.
- Un dataset pequeño `requirements.csv` y `tests.csv`.
- Una evaluación de retrieval con golden set.
- Una arquitectura dibujada de OpenWebUI -> LiteLLM -> vLLM -> Qwen.
- Un mini detector de anomalías CAN con autoencoder.
